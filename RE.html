Style El elemento HTML style contiene información de estilo para un documento o parte de un documento. S Contener CSS, que se aplica al contenido del documento que contiene el elemento style.
Ul El elemento HTML ul representa una lista desordenada de elementos, normalmente representada como una lista con viñetas.
  • El elemento ul es para agrupar una colección de elementos que no tienen un orden numérico, y su orden en la lista no tiene sentido. Normalmente, los elementos de la lista desordenada se muestran con una viñeta, que puede tener varias formas, como un punto, un círculo o un cuadrado. El estilo de viñeta no está definido en la descripción HTML de la página, pero en su CSS asociado, utilizando la propiedad list-style-type.
  • Los elementos ul y ol se pueden anidar tan profundamente como se desee. Además, las listas anidadas puede alternar entre ol y ul sin restricción.
  • Los elementos ol y ul representan una lista de elementos. Se diferencian en que, con el ol elemento, el orden es significativo. Para determinar cuál usar, intente cambiar el orden de la elementos de la lista; si se cambia el significado, se debe usar el elemento ol, de lo contrario, puede usar ul.
Li El elemento HTML li Debe estar contenido en un padre elemento: una lista ordenada ol, una lista desordenada ul o un menu. En menús y listas desordenadas, los elementos de la lista generalmente se muestran con viñetas. En las listas ordenadas, suelen ser mostrados con un contador ascendente a la izquierda, como un número o una letra. Se utiliza para representar un elemento en una lista.
Form El elemento HTML form representa una sección de documento que contiene controles interactivos para envío de información. Enviar informacion por un metodo POST por lo general
Input El elemento HTML input se utiliza para crear controles interactivos para formularios basados en web con el fin de aceptar datos del usuario; una amplia variedad de tipos de datos de entrada y widgets de control están disponibles, según el dispositivo y el agente de usuario. El elemento input es uno de los más potentes y complejo en todo HTML debido a la gran cantidad de combinaciones de tipos de entrada y atributos.

Tbody El elemento HTML body encapsula un conjunto de filas de tabla (elementos tr), lo que indica que comprende el cuerpo de la tabla (tabla).
  • Si la tabla incluye un bloque thead (para identificar semánticamente una fila de encabezados de columna) , el bloque tbody debe venir después de él.
  • Si los elementos tr se especifican fuera de un elemento tbody existente, como hijos directos del table , estos elementos serán encapsulados por un elemento tbody separado generado por el navegador.
  • Al imprimir un documento, los elementos thead y tfoot especifican información que puede lo mismo, o al menos muy similar, en cada página de una tabla de varias páginas, mientras que el tbody el contenido del elemento generalmente diferirá de una página a otra.
  • Cuando una tabla se presenta en un contexto de pantalla (como una ventana) que no es lo suficientemente grande para muestra toda la tabla, el agente de usuario puede permitir que el usuario desplace el contenido del thead , bloques tbody, tfoot y cantion separados entre sí para la misma tabla principal.
  • Puede utilizar más de un tbody por tabla, siempre que todos sean consecutivos. Esto le permite divida las filas en tablas grandes en secciones, cada una de las cuales puede formatearse por separado si es así deseable. Si no se marcan como elementos consecutivos, los navegadores corregirán este error de autor, asegurarse de que los elementos thead y tfoot se representen como el primer y el último elemento de la tabla, respectivamente.
