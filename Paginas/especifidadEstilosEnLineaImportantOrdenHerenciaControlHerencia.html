ESPECIFICAD

Podemos usar diferentes combinaciones de selectores para seleccionar un mismo elemento.

            [index.html]
            <p class="saludo"> HOLA MUNDO</p>
            [estilos.css]
            p{
                color: blue;
                background-color: yellow;
            }

            .saludo{
                color: green;
            }

¿En qué color se muestra el texto?

La especificidad es la manera mediante la cual los navegadores deciden el valor que será aplicado a una propiedad de un elemento.

Nota:

No se sobrescribe toda la regla, solo las propiedades que aparecen en ambas reglas.

¿Cómo calculan los navegadores la especificidad?

  1. Sean las dos combinaciones de selectores, que seleccionan el mismo elemento p:

    #id_1 .clase_1.clase_2 p{ color: green;}

    #id_1 .clase_1 p{color: red;}

    Para cada una de estas combinaciones el navegador calcula cual es más específica.

  2. Para cada combinación de selectores que seleccionan un elemento en particular (en este caso el elemento p), construimos una estructura del tipo a - b - c, denominada especifidad, donde a,b y c son enteros.

    1. El número a es la cantidad de veces que aparece un selector de id.

    2. El número b es la cantidad de veces que aparece un selector de clase, más la cantidad de veces que aparece un selector de atributo, más la cantidad de veces que aparece una pseudo-clase.

    3. El número c es la cantidad de veces que aparece un selector de tipo, más la cantidad de veces que aparece un pseudo-elemento.

      #id_1 .clase_1.clase_2 p{ color: green;} _________________(1- 2 - 1)

      #id_1 .clase_1 p{color: red;}_____________________________(1 - 1 - 1)

  3. Determinamos qué combinación de selectores es más específica. Para esto comparamos la especificidad asociada a cada combinación de elementos:

    1. La combinación más específica será aquella cuyo valor de a es mayor.
    2. En caso de que las combinaciones tengan el mismo valor a, la combinación más específica será aquella cuyo valor de b es mayor
    3. En caso de que las combinaciones tengan el mismo valor b, la combinación más específica será aquella cuyo valor de c es mayor.

  4. La combinación de selectores que sea más específica es la que gana.

    Nota:

    El selector universal (*), los operadores de combinación (+, >, ~, ' ' ) y el selector de ajuste de especificidad (:where()) no tienen efecto en el cálculo final.

    Las pseudo clases :not() y :is() no tienen efecto en la especificidad, pero sus parámetros sí. La contribución que se toma en cuenta es la del parámetro (es decir selector) con mayor peso.

ESTILOS EN LINEA

Los estilos en línea, es decir, las declaraciones CSS especificadas usando el atributo style, tienen prioridad sobre todos las declaraciones normales, sin importar la especificidad.

!important

El uso de !important en una declaración CSS sobreescribe cualquier declaración, incluso las declaraciones CSS especificadas usando el atributo style.

Nota:

Es útil saber que existe el indicador !important para que sepa qué es, cuando lo encuentre en el código de otras personas. Sin embargo, le recomendamos encarecidamente que nunca lo use a menos que sea absolutamente necesario. El indicador !important cambia la forma en que funciona una hoja de estilos, por lo que puede hacer que la depuración de problemas sea realmente difícil de resolver, especialmente en una hoja de estilo grande.

ORDEN

Podemos tener una combinación de selectores que tengan la misma especificidad:

[index.html]

<p>HOLA MUNDO</p>

[estilos.css] 

            p{
                color: red;
                background-color: yellow;	
            }
            p{
                color: blue;
            }

¿En qué color se muestra el texto?

Cuando se tiene la misma especificidad, el orden es importante. La declaración CSS que se define al ultimo, es la que se aplica

HERENCIA

Algunos valores de las propiedades CSS que se han establecido para los elementos padre los heredan los elementos hijo, pero otros no. 

            <div>
                UCB	
                <p>HOLA MUNDO</p>
            </div>
            div{
                color: red;
            }

Algunas propiedades no se heredan, por ejemplo la propiedad border. Si los elementos hijos heredarán la propiedad border, cada elemento hijo tendría un borde, y este probablemente sea un efecto que no deseamos.

Nota:

En las páginas de referencia de las propiedades CSS de MDN encontrarás un cuadro llamado DEFINICIÓN FORMAL con información técnica que enumera una serie de puntos sobre cada propiedad, incluyendo cuáles se heredan y cuáles no. Véase, por ejemplo, la definición formal de la propiedad color.

https://developer.mozilla.org/en-US/docs/Web/CSS/color

Ejemplo

            <ul class="main" >
                <li>Item One</li>
                <li>Item Two
                    <ul>
                        <li>2.1</li>
                        <li>2.2</li>
                    </ul>
                </li>
                <li>Item Three
                    <ul class="special">
                        <li>3.1
                            <ul>
                                <li>3.1.1</li>
                                <li>3.1.2</li>
                            </ul>
                            </li>
                        <li>3.2</li>
                    </ul>

            </ul>

                border: 2px solid #ccc;
                padding: 1em;
            }
            
            .special {
                color: black;
                font-weight: bold;
            }

Controlar la herencia

Cada propiedad css acepta 2 valores para controlar la herencia

inherit (Obligar a heredar)

La propiedad tendrá el mismo valor que la propiedad correspondiente del padre.

Ejemplo:

            <div>
                <p>HOLA MUNDO</p>
            </div>
            div{
                padding: 8px;
                color: red;
                border: 2px solid green;
            }

            p{
                border: inherit;
            }

initial (romper la herencia)

La propiedad tendrá el valor por defecto, es decir el valor establecido en la hoja de estilo por defecto del navegador.

Ejemplo:

            <div>
                PRIMERA CADENA
                <p>SEGUNDA CADENA</p>
            </div>
            div{
                color: red;
            }
            p{
                color: initial;
            }
